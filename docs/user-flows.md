### UI Considerations

- **Speed**: Checklist must be visible within 2 seconds of launch
- **Scannability**: Pass/fail status visible at a glance (color + icons)
- **Action proximity**: Fix buttons directly next to failed items
- **Launch prominence**: Launch button is the primary action
- **Profile switching**: Easy to switch, but not intrusive

### Edge Cases

| Scenario | Behavior |
|----------|----------|
| All checks pass immediately | Launch button enabled, green status |
| User launches with failures | Warning dialog: "Continue anyway?" |
| Check hangs/times out | Show timeout error, offer to skip |
| Device disconnected mid-check | Show disconnected status, offer refresh |

### UI Considerations

- **Progressive disclosure**: Don't overwhelm with all options at once
- **Smart defaults**: Pre-select likely choices based on game
- **Skip options**: Every step can be skipped and configured later
- **Edit later**: Wizard is for onboarding, not the only way to configure

### Alternative: Power User Mode

Users can skip the wizard and directly edit the YAML/JSON profile file.

```yaml
# ~/.rigready/profiles/dcs-fa18.yaml
name: "DCS F/A-18C"
description: "Full cockpit setup"
game: dcs-world
launch:
  executable: "{DCS_INSTALL}/bin/DCS.exe"
checks:
  - type: device
    name: "WinWing Throttle"
    vendorId: "0x4098"
    productId: "0x0001"
# ... etc
```
